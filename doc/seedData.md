# Seed Data Specification

## 1. 목적

초기 Seed Data는 서울시 버스 관제 시스템 MVP를 로컬 개발, Docker 통합 실행, 프론트엔드 연동에서 재현 가능한 상태로 시작하기 위한 기준 데이터다.

Seed Data는 Spring `CommandLineRunner` 기반 컴포넌트에서 주입하며, SQL 파일(`data.sql`)은 사용하지 않는다.

## 2. 실행 정책

* 실행 프로파일: `local`, `prod`
* 실행 시점: 애플리케이션 시작 시점
* 대상 테이블: `routes`, `stops`, `route_stops`, `buses`
* 제외 테이블: `bus_locations`, `bus_events`
* 중복 방지: 대상 테이블 중 하나라도 데이터가 있으면 전체 Seed 주입을 건너뛴다.

`bus_locations`, `bus_events`는 `bus-simulator` 또는 실제 차량 단말이 `POST /v1/internal/telemetry`로 전송한 텔레메트리를 `control-api`가 수신하면서 생성한다.

## 7. ID 정책

각 테이블의 PK는 DB identity 전략으로 생성한다. 빈 DB에서 최초 실행하면 저장 순서대로 ID가 생성되지만, 애플리케이션 로직과 문서는 고정 ID에 의존하지 않는다.
